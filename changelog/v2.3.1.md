# v2.3.1 — Bugfix：雙 AI 對比 Epoch 不終止

**日期：** 2026-04-26
**類型：** patch

---

## 主題

修復即時雙 AI 對比介面中，兩方都沒有命後 Epoch 永遠不推進的 bug。

---

## 根本原因

`update()` 中 `lives <= 0` 時先設 `state = 'dead'`，再用 `setTimeout(1200ms)` 轉為 `state = 'gameover'`。
但 `loop()` 每幀都呼叫 `loadState(gsA)` 把全域 `state` 蓋回快照中的 `'dead'`，
導致 `setTimeout` 的效果被覆蓋，`gsA.state` 永遠卡在 `'dead'`。
`checkDualEpoch()` 只判斷 `state === 'gameover'`，所以從不觸發，Epoch 永遠不推進。

---

## 修復

### Fix 1：`checkDualEpoch()`

改為也接受 `lives <= 0` 作為「此 instance 已死亡」的判斷條件：

```js
// 改前
if (gsA.state === 'gameover' && !gsA.epochDone) {
// 改後
if (!gsA.epochDone && (gsA.state === 'gameover' || gsA.lives <= 0)) {
```

`lives` 的值在 `saveState/loadState` 中正確快照，`update()` 執行 `lives--` 後下一幀即可偵測到。

### Fix 2：`loop()` 的 update 條件

`epochDone = true` 後停止呼叫 `update()`，避免繼續觸發碰撞邏輯：

```js
// 改前
if (gsA.state !== 'gameover') {
// 改後
if (!gsA.epochDone) {
```

draw 部分保持不動，死亡爆炸動畫繼續播放直到 Epoch 重啟。
